Introduction

Overview of the testing strategy

This chapter explains the testing strategy applied to the Suhad Hospital Management System. The system is a web-based hospital dashboard integrated with a DeepSeek AI engine and Firebase backend. Three separate testing techniques were utilized — Unit Testing, Integration Testing, and Usability Testing — to comprehensively assess the system's functionality, module interaction, and user experience.

✅ Summary of Usability Test Results

  • All tasks were completed successfully without errors or difficulties.
  • The Arabic RTL interface matches the participants' natural reading direction.
  • Color-coded status badges (حرج / تحذير / مستقر) enabled quick visual patient assessment.
  • The Quick Actions section provided fast access to procedures, medications, and labs.
  • The DeepSeek AI-generated Arabic medical report (arabicSummary) simplified complex clinical data for nursing staff.
  • No design changes were required following the usability test.
VI.4 Conclusion

Testing Phase Summary

This chapter detailed the testing phase of the Suhad Hospital Management System using three complementary methods. Unit testing confirmed that all individual components handle both valid and invalid inputs correctly. Integration testing verified that data flows seamlessly between the web dashboard and Firebase backend, including DeepSeek AI report generation and FCM push notification delivery triggered only on critical patient status. Usability testing with real hospital staff confirmed that the system is intuitive, efficient, and ready for clinical deployment.
